## Key Design Decisions

### Anti-Leakage Protocol
- `StandardScaler` is always inside `sklearn.Pipeline` — fitted only on training folds
- Data augmentation (Gaussian ±5%) is applied **exclusively within the training fold**
  of each outer CV split; test folds always contain original unperturbed samples only
- OOF meta-features for stacking are generated via inner 4-fold CV within the training
  partition — the test fold is never seen until final prediction

### Data Augmentation
`src/augmentation.py` exposes `augment_train_fold(X_tr, y_tr, seed)` which is called
inside the CV loop in `evaluation.py`. It is **never** called on the full dataset before
splitting. This ensures no augmented variant of a base sample can appear in a test fold
simultaneously with its source.
